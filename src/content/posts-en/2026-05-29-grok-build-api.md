xAI released "grok-build-0.1," the foundation model behind the coding agent "Grok Build," as a public beta on the xAI API. The key point is that developers can now embed the model directly into their own tools and frameworks without needing a SuperGrok or X Premium Plus subscription.

## Details

- **What the model is**: The same model that powers the Grok Build CLI under the hood, trained specifically for agentic coding tasks such as web development, debugging, and MCP support
- **Pricing**: Low pricing at $1 per million input tokens and $2 per million output tokens
- **Speed**: Delivers fast response performance exceeding 100 tokens per second
- **Broader use cases**: Positioned as an economical option for agentic tool-calling use cases beyond coding as well
- **Ecosystem support**: Available via the core xAI API as well as OpenRouter and the Vercel AI Gateway. Integrations are also progressing with external frameworks such as Cursor, Hermes Agent, OpenClaw, Kilo Code, and OpenCode

## What happened next

- On June 11, a plugin marketplace was added to Grok Build itself
- On June 15, an "Agent Dashboard" was added for managing multiple sessions at once
- On July 15, in response to a recently uncovered issue involving unauthorized repository uploads, the entire Grok Build source code was released under the Apache 2.0 license
