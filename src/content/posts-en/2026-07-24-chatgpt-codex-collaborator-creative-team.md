OpenAI published an internal case study on Chad Nelson, a creative specialist at the company, who uses its AI coding agent Codex to build demos and creative workflows for brand partners. The takeaway: Codex is moving beyond its role as a developer tool and starting to function as a "collaborator" for non-engineering creative professionals too.

## Details

- **Campaign output**: Normally producing 5-10 campaign directions a day, Nelson used Codex to generate 50 in a single day, then narrowed them down to the 10 strongest to pitch to clients
- **Custom UI building**: Built bespoke interfaces with sliders and controls to visualize camera composition and lighting adjustments in real time
- **Workflow automation**: Assembled workflows that wire together API integrations using only natural-language instructions, prototyping without needing to write code
- **Context awareness**: Codex can factor in project briefs, brand books, and composition or tone requirements — a distinction the article draws from standard creative software
- **What this isn't**: Not a new feature or plan announcement — this is a case study highlighting internal usage of Codex

## How to try it

- Codex is already available through existing ChatGPT Plus/Pro/Team/Enterprise plans via CLI, IDE extensions, and cloud agents — no new plan or feature is being introduced here
- See OpenAI's official blog post for the full workflow details and case study
