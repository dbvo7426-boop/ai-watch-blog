Cursor has updated its Slack agent integration. The agent now presents a plan before starting work, allowing users to redirect it early on, and it now supports multi-repo environments as well as work that spans other Slack channels.

## Details

- **Plans shown before work starts**: The agent now "presents a plan before diving in," letting users intervene and adjust course early. Progress is also shown in real time as each step happens
- **UI refresh**: Buttons in messages have been replaced with compact footer links, and tables, pull requests, and other outputs are now displayed more clearly
- **Support for multi-repo environments**: The agent can now be launched not just against a single default repository but within named, multi-repo environments. It identifies when a task requires access across multiple repos — such as frontend, backend, and shared code — and selects the right environment, with a dedicated button to switch repos mid-task
- **Cross-channel workflows**: The agent can now read information from, and send messages to, other Slack channels and threads. It gathers context from elsewhere in the workspace while working, and posts updates back to the original thread or related channels as needed

## How to try it

This is a standard update rolling out to existing users of Cursor's Slack integration. Detailed setup instructions are available in Cursor's official Slack integration docs (cursor.com/docs/integrations/slack).
